Fire Safety Risk Assessment Expert System
University DRLP Project Specification
Project Goal

Develop a Fire Safety Risk Assessment Expert System using SWI-Prolog.

The system must simulate the reasoning process of a fire safety inspector.

The system should not merely store data. It must infer conclusions by applying fire safety knowledge and rules to user-provided building information.

Expert System Architecture

The system must clearly contain:

Knowledge Base
        +
Inference Engine
        +
User Facts
        =
Expert System

Reasoning Flow:

User Inputs
      ↓
Building Facts
      ↓
Knowledge Base
      ↓
Inference Rules
      ↓
Violation Detection
      ↓
Recommendations
      ↓
Safety Score
      ↓
Risk Level
      ↓
Compliance Status

Knowledge Base

The system must contain explicit fire safety knowledge.

Exit Requirements
exit_requirement(50,1).
exit_requirement(100,2).
exit_requirement(300,3).
exit_requirement(99999,4).

Interpretation:

Occupancy ≤ 50 → 1 exit

Occupancy ≤ 100 → 2 exits

Occupancy ≤ 300 → 3 exits

Occupancy > 300 → 4 exits
Staircase Requirements
stair_requirement(2,1).
stair_requirement(5,2).
stair_requirement(99999,3).

Interpretation:

Floors ≤ 2 → 1 staircase

Floors ≤ 5 → 2 staircases

Floors > 5 → 3 staircases

Equipment Knowledge
equipment(alarm_system,critical).
equipment(sprinkler_system,critical).
equipment(emergency_lighting,important).
equipment(evacuation_maps,recommended).
equipment(fire_resistant_doors,recommended).
Risk Weight Knowledge
risk_weight(insufficient_exits,25).
risk_weight(insufficient_staircases,20).
risk_weight(alarm_missing,20).
risk_weight(sprinkler_missing,15).
risk_weight(lighting_missing,10).

Risk score should use these weights instead of a fixed deduction.

Recommendation Knowledge

Store mappings between violations and corrective actions.

recommendation_for(
    insufficient_exits,
    'Increase number of emergency exits'
).

recommendation_for(
    insufficient_staircases,
    'Provide additional staircases'
).

recommendation_for(

).

recommendation_for(
    sprinkler_missing,
    'Install an automatic sprinkler system'
).

recommendation_for(
    lighting_missing,
    'Install emergency lighting'
).
Building Type Specific Rules
Hospital Rules

Hospitals always require sprinklers.

Violation:

Hospital Requires Sprinkler System
Factory Rules

Factories are considered high-risk buildings.

Factories must have:

Alarm System
Sprinkler System

Generate mandatory issues if either is missing.

Hotel Rules

Hotels with occupancy above 150 require:

Minimum 3 exits

Violation:

Hotel Occupancy Requires Additional Exits
School Rules

Schools with occupancy above 200 require:

Minimum 2 staircases

Violation:

School Requires Additional Staircases

Safety Score

Start with:

100

Subtract violation weights.

Example:

Alarm Missing = -20

Insufficient Exits = -25

Lighting Missing = -10

Final Score:

45

Minimum:

0

Use:

findall/3

to gather violations.

Risk Levels
80 – 100  → LOW

50 – 79   → MEDIUM

0 – 49    → HIGH
Compliance Status

PASS

when:

\+ mandatory_issue(...)

FAIL

when one or more mandatory issues exist.
